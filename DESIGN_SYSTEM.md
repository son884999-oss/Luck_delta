# 천문 — Design System Specification v1 (Signature / Digital Heritage)

> Tasks 7–12 + 홀리스틱 UI/UX 최적화를 하나의 시스템으로 통합한 명세.
> 목표: **모든 세대(특히 고령층)에게 직관적이고 고급스러운, 정보과부하 없는** 디지털 운세 서비스.

## 0. 팔레트 결정 (⚠️ 방향 전환 — 확인 필요)

기존 확정 방향은 **보석 멀티톤 / #07060d**. Task 9는 **Deep Navy & Warm Gold**를 요구 → 두 방향이 충돌.
**제안(절충, 권장):**
- **브랜드/크롬 = Deep Navy + Warm Gold** — 배경을 순흑(#07060d)에서 **딥 네이비(#0A1024 → #070A16)** 로 미세 이동, **Warm Gold(#C9A24B/#E7B94F)** 를 신뢰·시그니처의 주 강조색으로 격상.
- **보석 멀티톤은 '기능 색상'으로만 잔존** — 6개 운세 모드의 카테고리 식별(인디고/바이올렛/앰버/로즈/에메랄드/스카이)에 한정. 오행 색도 유지(데이터 의미).
- 이렇게 하면 Task 9의 "신뢰·우아함"과 기존의 "카테고리 색 코딩"을 모두 보존.
- **전면 리페인트(보석톤 폐기)는 대규모 작업이므로 착수 전 승인 필요.** 아래 스펙은 이 절충 팔레트 기준.


## 1. 타이포그래피 — 엄격한 3단 위계

노이즈 제거를 위해 **딱 3개 크기만** 사용(+캡션 1개 예외).

| 토큰 | 크기/굵기 | 용도 | 대비 기준 |
|---|---|---|---|
| **Header** | 26–34px / 900 Serif(Noto Serif KR) | 화면 제목, 핵심 숫자(점수) | AAA(큰글씨 4.5:1↑) |
| **Sub** | 17–18px / 700 Sans | 카드 제목, 섹션명 | AAA 7:1↑ |
| **Body** | 15–16px / 400–500 Sans, line-height 1.7 | 본문·설명 | AAA 7:1↑ |
| (Caption) | 13px / 700 | 라벨·뱃지(최소화) | AAA |

**규칙:** 같은 화면에 4개 이상 크기 금지. 10–12px 저대비 회색 라벨 폐기(고령층 가독성). 폰트크기 토글은 **보조 선택지**로만 — 기본값이 이미 충분히 큼.

---

## 2. 여백(Negative Space) — 숨 쉬는 레이아웃

- 카드 간 간격 **20–24px**(기존 12–16 → 확대). 카드 내부 padding **18–20px**.
- 화면 좌우 여백 **20–24px** 유지. 요소 사이 '비움'으로 그룹을 시각 분리(구분선 대신 공간).
- 한 화면에 **주요 액션 1개 + 보조 2~3개**만 노출. 나머지는 **Progressive Disclosure**.

---

## 3. Progressive Disclosure (정보 위계)

| 항상 보임(1차) | 펼쳐서 보임(2차, '더보기'/카드 진입) |
|---|---|
| 내 정보 카드(이름·생년월일·띠) | 일주·오행 상세 → 사주카드/평생사주 |
| 시그니처 리포트 진입 | 리포트 포함 내용 → 신청 화면 |
| 오늘의 운세 1개 + 다이어리 | 지난 기록 전체 → 일기 허브 |
| 일기 2줄 미리보기 | 전체 → `ExpandableText` 더보기 |
| 발송 상태바(준비 중 N분) | 상세 → 신청 완료 화면 |

이미 적용: `ExpandableText`(2줄+더보기), 일기 허브 분리, 명식 상세는 사주 모드로.

## 4. 인터랙션 철학

- **OS 네이티브 alert 전면 금지 → 커스텀 `ConfirmDialog`** (글래스·고대비·큰 버튼·Esc/바깥탭·`role=dialog`). 적용 완료: 일기 삭제, 정보 초기화.
- **선형 내비게이션:** 모든 결과/하위 화면의 1차 행동은 **'메인으로(Return to Main)'**. 레이아웃이 복잡해지면 **'다시 보기' 등 보조 버튼 제거**.
- **마이크로 인터랙션:** fade-up/scale-in 등장, 탭 시 `active:scale`, 점수 링 카운트업. 과하지 않게.

## 6. "60세·비숙련 사용자"의 인지부하를 어떻게 줄이나

- **'무엇을 누를지 모름' 해소:** 화면당 1차 행동을 1개로 한정 + 골드 글로우로 시그니처를 시각적 앵커로. 시선 흐름: 내 정보 → 시그니처 → 오늘의 운세.
- **읽기 부담 해소:** 기본 16px 본문·7:1↑ 대비·1.7 줄간격. 시진 등 전문용어 제거.
- **실수 불안 해소:** 삭제·초기화는 명확한 커스텀 확인 모달 + '되돌릴 수 없음' 고지.
- **선택지 과부하 해소:** 2차 기능은 접어두고 필요 시 펼침(Progressive Disclosure).

## 7. 페르소나 사용성 테스트 (Task 10) + v2.0 조정

**User A (대학생·미니멀 선호):**
- 마찰: ① 허브에 카드가 많아 스크롤이 길다 ② 시그니처가 하단이라 한 번에 안 보임 ③ '다시 보기'가 군더더기.
**User B (50대·가독성 우려):**
- 마찰: ① 비선택 휠 항목이 흐려 날짜 고르기 어려움(→ Task5에서 개선됨) ② 작은 회색 라벨 ③ 네이티브 팝업이 앱과 동떨어져 당황.
**User C (그래픽 디자이너·위계 비평):**
- 마찰: ① 폰트 크기 종류가 너무 많음 ② 여백 부족으로 답답 ③ 골드/보석톤 혼재로 위계 불명확.

**→ Version 2.0 UX Adjustments**
1. **시그니처 최상단 이동** + 6모드는 일일 2개만 1차 노출, 나머지 '더 보기' 접기 (A·C).
2. **3단 폰트 위계 엄수**, 10–12px 저대비 라벨 폐기 (B·C).
3. **카드 간 20–24px 여백** 표준화 (C).
4. **네이티브 alert → 커스텀 모달** (B, 적용 완료).
5. **결과화면 '다시 보기' 제거, '메인으로' 단일 1차 버튼** (A·C).
6. **휠/라벨 기본 고대비·대형화** (B, Task5 적용 완료).

## 8. 시그니처 발송 워크플로우 (Task 7 + 8) — 로직 & 샘플

### 흐름
```
[이메일 입력] → [요약 확인 모달] → (확인) → [발송 능력 평가] → [카운트다운 + 진행바] → [발송]
```

1. **요약 확인 모달**(커스텀): 입력값 재노출
   - 생년월일: 2004년 11월 3일 / 태어난 시각: 새벽 3시 13분 / 이메일: hmcf7nz7@naver.com
   - "이 정보가 맞나요?" → [수정] / [맞아요, 신청]
2. **적응형 발송 평가** `assessDispatchEta()`:
   ```
   대기열(최근 10분 내 미발송 요청) 있음?  ── 예 ──▶ 10분 (적체)
            │ 아니오
   피크 시간대(심야 22–01, 점심 12–14)?  ── 예 ──▶ 10분
            │ 아니오
   랜덤 혼잡도 < 0.25 ?                  ── 예 ──▶ 10분
            └─────────────────────────── 아니오 ─▶ 5분
   ```
   (백엔드 부재 → 클라이언트 시뮬레이션. 서버리스 도입 시 실제 큐 길이/응답시간으로 대체.)
3. **저장** `localStorage.cm_report_pending = { email, birth, sendAt: now + eta*60000, etaMin, createdAt }`.
4. **명확한 도착 안내(샘플 출력):**
   - 5분: "시그니처 리포트를 **오전 9시 18분**에 보내드릴게요. (약 5분 후)"
   - 10분(혼잡): "이용자가 많아 조금 더 정성껏 준비 중이에요. **오전 9시 23분**에 도착해요. (약 10분 후)"
5. **허브 상태바(진행 표시):** "시그니처 준비 중 · 4분 남음" + 진행 바(경과/총 시간). 도착 시각 도달 → "발송 완료 ✦".

### 테스트 케이스 (요청 입력)
```
입력  : email=hmcf7nz7@naver.com, 2004-11-03, 03:13(새벽)
명식  : 갑신·갑술·병술·기축 (검증된 계산)
평가  : 비피크 가정 → 5분
저장  : sendAt = 신청시각 + 5분
안내  : "시그니처 리포트를 [신청+5분]에 보내드릴게요."
허브  : 카운트다운 5→0분, 진행바 0→100%, 이후 "발송 완료"
```
> *상태*: ✅ **구현 완료** — 요약 확인 모달(`ConfirmDialog`)·적응형 ETA(`assessDispatchEta`)·완료 화면 도착 안내·허브 진행바(`DispatchStatusBar`) 모두 적용 및 Playwright 검증. 단, **실제 메일 발송은 서버리스(F2) 도입 후 라이브** — 현재 '발송 완료'는 카운트다운 종료를 표시하는 클라이언트 시뮬레이션.

## 9. 궁합 관계 아키타입 (Task 12) — 적용 완료

시간 기반('대길의 날/흉') → **관계의 결** 묘사로 교체(채점 로직 유지, 라벨만).

| 점수대 | 아키타입 | 의미 |
|---|---|---|
| ~62 | 마주 보는 거울 | 닮음과 다름을 비추며 알아가는 사이 |
| ~74 | 잔잔한 동행 | 시간이 쌓일수록 깊어지는 인연 |
| ~86 | 서로의 빈자리 | 부족함을 채워주는 보완의 힘 |
| ~94 | 단단한 매듭 | 안정·신뢰로 결이 맞는 조화 |
| ~100 | 운명의 한 쌍 | 깊은 이해와 끌림이 함께하는 인연 |

**가치:** '오늘 운 좋음' 같은 1회성 시간 라벨과 달리, 두 사람 **관계의 본질**을 규정해 기억·공유·재방문 가치가 큼. 시그니처 브랜딩(통찰·품격)과 일관.

## 10. 시그니처 카드 비주얼 에셋 (Task 11)

- **하이라이트(적용):** 네온 X. **부드러운 골드 외부 글로우 + 내부 상단 하이라이트(이너 스트로크)** 로 '빛이 스며드는' 차분한 프리미엄감.
  `box-shadow: 0 10px 40px rgba(231,185,79,.10), inset 0 1px 0 rgba(255,255,255,.09)`.
- **권장 로열티프리 에셋 스타일:** ① **추상 골드 텍스처**(은은한 금박·빛 번짐) ② **수묵/먹 번짐(ink-wash)** 미니멀 ③ **고전 종이 결(paper texture)**. 출처: PD/CC0(예: 공공도메인 텍스처, 자체 SVG 그라데이션 권장 — 인쇄/리포트와 동일하게 자체 SVG가 가장 안전·일관).
- **블렌딩:** 딥 네이비 위에 **`mix-blend-mode: soft-light`·낮은 opacity(8–14%)** 로 골드 텍스처를 얹어 카드 우측 상단에만 은은히. 텍스트 영역은 텍스처를 피해 가독성 확보. 결과: "이게 핵심 고급 기능"임을 공격적이지 않게 전달.
