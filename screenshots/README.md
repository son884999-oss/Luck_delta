# 천문 — 기능별 화면 스크린샷 & 발표자료

각 기능 화면을 캡처해 변경 전후 비교·기록용으로 보관하고, 그대로 발표자료(PPTX)를 자동 생성합니다.

## 한 번에 (촬영 → 최신화 → 발표자료 빌드)
```bash
npm run deck                  # 캡처 + 발표자료 + 파이프라인 설명서까지 한 번에
```

## 개별 명령
```bash
npm run shots                 # 화면+상호작용 캡처(배포 사이트 기준)
npm run shots http://localhost:5173   # 로컬(dev 서버) 기준
npm run pptx                  # screenshots/ 이미지로 천문_발표자료.pptx 생성
npm run pptx:pipeline         # 천문_자동화_파이프라인.pptx(자동화 로직 설명) 생성
```
시스템 Edge(Playwright `channel:'msedge'`)로 모바일 뷰(412×915, 2x)에서 캡처합니다.
앱은 URL 라우팅이 없어 localStorage(`cm_birth`)를 시드해 허브로 부팅한 뒤 각 기능으로 진입합니다.
상호작용(카드 뒤집기 전/후, 점수 공개, 타로 뽑기→결과, 궁합 입력→결과, 천문식탁 추천+주변식당 등)까지 캡처합니다.

## 폴더 구조
- `NN_*.png` — 스크립트가 자동 촬영한 화면(덮어쓰기 갱신)
- `u_*.{jpg,png}` — 사용자가 제공한 실제 폰 캡처를 발표용으로 정리한 사본(비공개/폐기 기능 등)
- `user/` — 사용자가 원본 그대로 넣어둔 에셋 보관함(스크립트가 건드리지 않음)
- `28_1_report_real.png` — 실제 PDF 리포트 캡처(사용자 제공, 발표 핵심 자료)

## 산출물(PPTX)
- **천문_발표자료.pptx** — 공문서 양식 23슬라이드. 표지 + 기능 17 + 비공개 기능 3 + 기술/종합.
- **천문_자동화_파이프라인.pptx** — 위 자동화 로직(캡처→최신화→빌드)을 설명하는 8슬라이드 부속 자료.
- 대본 메모: `발표자료_내용.md`

> 비공개/폐기 기능(식약처 음식 검색·컴플라이언스, 이메일 리포트)도 사용자 제공 캡처로 슬라이드에 포함됩니다.
